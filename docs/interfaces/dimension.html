Interface Dimension

Data dimension defined in Config.dimensions

Dimensions are only meaningful within the context of a Schema. A Schema is in essence simply a set of dimensions that describe the shape of the data.

A dimension defined here that is not referenced in any schema will not be used in the tool. We define dimensions separately from schemas so that the same dimension may be used in multiple schemas without having to be defined multiple times.

Name used internally as a unique identifier for the dimension. Other parts of the config, like Schema.dimensions, can reference a dimension by this name.

Must be unique and should ideally be as short as possible without being cryptic

options

options: Array<string | number | Option>
  • Defined in config.spec.d.ts:224

Array of possible values for the dimension

The Option type allows specifying the (unique) internal name and the name used for display separately. Options may also be specified simply as numbers or strings. In that case the same value (which must be unique) will be used both internally and for display.

Optional playable

playable: undefined | false | true
  • Defined in config.spec.d.ts:259

Enable the playback feature for this dimension?

If enabled, we render a play button next to the dimension control. Pressing play begins a timed sequence of state changes in which we step through the array of options for this dimension.

default: false
