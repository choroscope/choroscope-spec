Optional fieldnames

fieldnames: AdminFieldnames[]
  • Defined in config.spec.d.ts:262

Names of the metadata fields for each feature in the administrative shapefiles

The index in the array corresponds to the admin level, so the first item represents fields for admin0, the second fields for admin1, etc.

default:

[
  { "id": "ADM{N}_CODE", "name": "ADM{N}_NAME" },
  ...
]

where {N} represents each admin level from 0 up to the max admin level

Optional filepaths

filepaths: string[]
  • Defined in config.spec.d.ts:245

Relative paths to the administrative shapefiles.

The index in the array corresponds to the admin level, so the first item represents admin0, the second admin1, etc.

default:

["shapefiles/admin{N}/admin{N}.shp", ...]

where {N} represents each admin level from 0 up to the max admin level
