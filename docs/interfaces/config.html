Optional filepath_raster_mask

filepath_raster_mask: undefined | string
  • Defined in config.spec.d.ts:52

Relative path to a GeoTIFF file to be applied as mask to each GeoTIFF data file

Example: "data/raster/mask.tiff"

The process of masking compares each data file, pixel by pixel, to the mask. If the mask contains a "no data" value for a given pixel, the corresponding pixel in the output raster will also have a "no data" value. Otherwise it will have the value from the corresponding pixel in the data file.

If this field is absent, no masking will occur.
